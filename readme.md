# WeatherPredictor — 智能天气预测器

## 产品概述

WeatherPredictor 是一款基于 **ESP8266** 的智能桌面天气站，集成 OLED 显示、语音播报、室内温湿度传感和 MQTT 远程控制功能。设备自动从心知天气 API 获取实时天气和 3 日预报，通过 1.3 英寸 OLED 屏幕直观展示，并支持语音播报天气信息。

配套旋钮翻页遥控器，通过 MQTT 协议远程翻页，无需接触主设备。

## 一、主设备 — WeatherPredictor

### 1. 配网系统 (WiFiManager)

- 首次上电自动进入**配网模式**，OLED 显示提示信息
- 手机/电脑连接热点 `AutoConnectAP`，自动弹出配置 portal
- 可配置参数：
  - **城市**（默认 `beijing`）
  - **MQTT 服务器地址**（默认 `192.168.5.4`）
- 配置保存后自动重启连接 Wi-Fi

### 2. 天气数据

- 数据源：[心知天气 API](https://www.seniverse.com/)（HTTPS + BearSSL）
- **当前天气**：温度、天气状况文字、天气代码
- **3 日预报**（今日/明日/后天）：最高温、最低温、白天/夜间天气、降水概率、风向、风力
- 自动更新间隔：
  - 当前天气：**10 分钟**
  - 未来天气：**6 小时**
- API 请求失败时最多重试 2 次，仍失败则发布失败事件到 MQTT

### 3. 页面展示

| 页码 | 内容 |
|:---:|------|
| 0 | 当前天气（大号温度 + 天气图标 + 室内温湿度） |
| 1 | 今日预报（白天/夜间天气、温度范围、降水、风力） |
| 2 | 明日预报 |
| 3 | 后天预报 |

- 状态栏统一显示：**当前时间**、**城市名称**、**语音状态**（VOC/MUT）
- 天气图标根据"晴/多云/雨/雪/阴"自动切换

### 4. 按键操作

| 按键 | GPIO | 功能 |
|------|:---:|------|
| 电源开关 | 16 | 切换开机/关机；开机自动开启语音，关机关闭屏幕和语音 |
| 翻页 | 14 | 循环切换 4 个页面 (0→1→2→3→0)，重置自动翻页计时 |
| 静音 | 12 | 切换语音播报 开启/关闭 |
| 播报 | 15 | 触发当前页面语音播报一次 |

### 5. 自动翻页

- **本地模式**下，页面每 **20 秒**自动切换一次
- 手动翻页后计时器**重置**
- 关机或**远控模式**下自动翻页暂停

### 6. MQTT 通信

#### 主题结构

```
clock/equip1/
├── state              — 设备状态：mode, page, rssi, voicestate
├── state/sensor       — 传感器数据：temperature, humidity
├── state/city         — 当前城市
├── set                — 设置命令
├── control            — 控制命令
└── event              — 事件通知
```

#### 控制命令 `clock/equip1/control`

```json
{"command": [2, null, null, null, null]}
```

| 索引 | 字段 | 类型 | 说明 |
|:---:|------|------|------|
| 0 | page | int | 跳转到指定页面 (0-3) |
| 1 | update_today | bool | 强制刷新当前天气 |
| 2 | update_future | bool | 强制刷新未来天气 |
| 3 | voice_trigger | bool | 触发语音播报 |
| 4 | voice_state | bool | 设置语音开关 |

> 不需要的字段传 `null`，服务端通过 `isNull()` 跳过，不会触发任何操作。

#### 设置命令 `clock/equip1/set`

```json
{"command": ["shanghai", true]}
```

| 索引 | 字段 | 类型 | 说明 |
|:---:|------|------|------|
| 0 | city | string | 修改城市（同步更新 API URL 并刷新天气） |
| 1 | remote_mode | bool | `true`=远程模式，`false`=本地模式 |

#### 模式说明

| 模式 | 按键 | 自动翻页 | MQTT 控制 | 开关机 | 旋钮翻页 |
|------|:---:|:---:|:---:| :---:| :---:|
| 本地模式（默认） | 可用 | 可设置城市、控制模式 | 可用 | 可用 | 不可用 |
| 远程模式 | 不可用 | 不可用 | 可设置城市、控制模式、页面、语音播报、语音状态 | 可用 | 可用 |

### 7. 室内传感器

- DHT11 温湿度传感器，每 **10 秒**读取一次
- 数据同时显示在 OLED 首页和 MQTT `state/sensor` 主题

### 8. 时间同步

- NTP 服务器：`ntp.aliyun.com`、`cn.ntp.org.cn`，时区 UTC+8
- 启动时等待同步成功后才加载天气数据

## 二、遥控器 — RemoteControl

### 功能

通过旋转电位器远程切换主设备的页面。将电位器分为 4 个区间，旋到新区间时自动发送 MQTT control 消息（仅携带翻页信息，其余字段为 `null`）。

| 电位器 ADC 值 | 对应页面 |
|:---:|:---:|
| 0 ~ 255 | 第 0 页（当前天气） |
| 256 ~ 511 | 第 1 页（今日预报） |
| 512 ~ 767 | 第 2 页（明日预报） |
| 768 ~ 1023 | 第 3 页（后天预报） |

### 配网

- 上电后创建 AP 热点 **`RemoteSetup`**（与主设备的 `AutoConnectAP` 区分）
- 手机连接后弹出配置页面，可填写：
  - **WiFi 凭据**
  - **MQTT 服务器地址**（默认 `192.168.5.4`）
- 配网超时 180 秒，超时后自动重启

### 消抖机制

- 每 **100ms** 读取一次电位器
- 仅当页面区间发生变化时才发送 MQTT 消息，避免 ADC 抖动导致反复发送

## 三、接线图

### 主设备 — WeatherPredictor

| ESP8266 引脚 | 连接设备 |
|:------------:|----------|
| GPIO13 (D7)  | DHT11 DATA |
| GPIO14 (D5)  | 翻页按键（按下 → GND） |
| GPIO15 (D8)  | 播报按键（按下 → GND） |
| GPIO12 (D6)  | 静音按键（按下 → GND） |
| GPIO16 (D0)  | 电源按键（按下 → GND） |
| GPIO1 (TX)   | 语音模块 RX |
| SDA (D2)     | OLED SDA |
| SCL (D1)     | OLED SCL |
| 3.3V / GND   | DHT11、OLED、语音模块供电 |

> 按键均为下降沿触发（按下接 GND，内部上拉）。

### 遥控器 — RemoteControl

| ESP8266 引脚 | 连接设备 |
|:------------:|----------|
| A0           | 电位器中脚（信号） |
| 3.3V         | 电位器左脚 |
| GND          | 电位器右脚 |

> 使用 10kΩ 线性电位器即可。
